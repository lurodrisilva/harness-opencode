{{- if and .Values.auth.enabled (not .Values.auth.existingSecret) }}

The chart auto-generated a {{ .Values.auth.generatedPasswordLength }}-char password.
Retrieve it with:

  kubectl --namespace {{ .Release.Namespace }} get secret {{ include "opencode.authSecretName" . }} \
    -o jsonpath='{.data.{{ .Values.auth.passwordKey }}}' | base64 -d

The password persists across upgrades (chart uses lookup-then-fallback).
Rotate it explicitly by deleting the Secret and running `helm upgrade`.
{{- end }}

Ingress is enabled. Hostnames:
{{- range .Values.ingress.hosts }}
  https://{{ .host }}
{{- end }}

opencode speaks plain HTTP — your Ingress controller MUST terminate TLS for
every host above. Verify with:

{{- if and .Values.auth.enabled (not .Values.auth.existingSecret) }}
  PASSWORD=$(kubectl --namespace {{ .Release.Namespace }} get secret {{ include "opencode.authSecretName" . }} \
    -o jsonpath='{.data.{{ .Values.auth.passwordKey }}}' | base64 -d)
  curl -fsS -u {{ .Values.auth.username }}:$PASSWORD https://<host>/global/health
{{- else if .Values.auth.enabled }}
  # NOTE: Secret "{{ .Values.auth.existingSecret }}" must contain key "{{ .Values.auth.passwordKey }}".
  # Override `auth.passwordKey` if your BYO Secret uses a different key name.
  PASSWORD=$(kubectl --namespace {{ .Release.Namespace }} get secret {{ .Values.auth.existingSecret }} \
    -o jsonpath='{.data.{{ .Values.auth.passwordKey }}}' | base64 -d)
  curl -fsS -u {{ .Values.auth.username }}:$PASSWORD https://<host>/global/health
{{- else }}
  curl -fsS https://<host>/global/health
{{- end }}
{{- else }}

  kubectl --namespace {{ .Release.Namespace }} port-forward svc/{{ include "opencode.fullname" . }} 4096:{{ .Values.service.port }}

Then in another terminal:
